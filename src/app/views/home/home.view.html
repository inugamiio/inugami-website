Inugami

The Enterprise Observability Framework

Inugami is a high-performance, non-intrusive Java framework designed to bridge the gap between technical monitoring and business intelligence.

It provides a robust architecture to capture, enrich, and export critical execution data without impacting the end-user experience.

Core Philosophy: "Observe Everything, Impact Nothing"

Inugami is built on the principle that monitoring should be a transparent "sidecar" to your busines logic. By leveraging asynchronous processing and optimized data structures, it ensures that deep observability never comes at the cost of latency.

Key Features

Asynchronous KPI Extraction
Extract complex business indicators from requests and responses using the KpiExtractorSPI. Processing is decoupled from the main thread, ensuring lightning-fast response times.
Contextual Intelligence (MDC)
The MdcService provides a typed, structured way to manage the Mapped Diagnostic Context. It automatically correlates technical metadata (TraceIDs, spans) with business data (Order IDs, customer profiles).
Extensible via SPI
Inugami is designed for customization. Whether itâ€™s dynamic field injection (MdcDynamicFieldSPI) , custom type mapping, or sophisticated interception filtering (Interceptable), the framework adapts to your specific architecture.

Why Choose Inugami?

Unified Source of Truth
Merge technical health and business performance into a single dashboard.
Developer Friendly
Clean APIs and SPIs mean your business code remains focused on logic, not logging.
Production Ready
Built-in protection against memory leaks (MDC cleanup) and performance bottlenecks.
Flexible Storage
While optimized for modern log aggregators like Elasticsearch/Kibana, its structured output can be adapted to any monitoring stack.
Inugami is designed as a first-class citizen of the Spring Boot ecosystem

It leverages the core principles of Spring to minimize boilerplate code and automate infrastructure setup.
